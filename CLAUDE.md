# The Earth App - Website

Marketing landing page for **The Earth App** ("Explore with real people", the anti-doomscroll). Static-generated Nuxt 4 site served at `https://earth-app.com`, deployed to GitHub Pages.

This is the marketing site only. The actual web app lives in the sibling `crust` repo (deployed to `app.earth-app.com`); the mobile app is `sky`. This repo just redirects into those and links out. crust serves its own sitemap/robots for `app.earth-app.com` - don't try to cover app routes from here.

## Stack

- Nuxt 4 (`srcDir: 'src'`), Vue 3, SSG (`nitro.preset: 'static'`) prerendered to `.output/public` / `dist`.
- Tailwind CSS v4 (via `@tailwindcss/vite`), `@nuxt/ui` v4, `@nuxt/icon` (css mode), `@nuxt/image`.
- Bun is the package manager and CI runner (`oven-sh/setup-bun`). Not npm.

## Commands

- `bun run dev` - dev server (`nuxi dev --public`)
- `bun run build` / `bun run generate` - static build (CI runs `build`, uploads `./dist` to Pages)
- `bun run prettier` / `bun run prettier:check` - format (CI gates on `prettier:check`; husky + lint-staged run it on commit)

## Layout

- `src/pages/index.vue` - the only real page; everything else in `routeRules` is a 301 redirect to a subdomain (`app.`, `api.`, `blog.`, `support.`) or the App Store.
- `src/components/` - top-level components (`NavBar`, `Primary`) live in the root; grouped components use the shorthand name inside their folder (`onboarding/Quest.vue` -> `<OnboardingQuest>`).
- `src/app.vue` - global `useSeoMeta` (OG/Twitter/Apple) + `useSchemaOrg` (WebSite + SoftwareApplication).
- `src/app.config.ts` - `name`, `description`, `themeColor` (`#43b54d`).

## SEO configuration

All four modules share the site URL from `nuxt-site-config`. `NUXT_PUBLIC_SITE_URL` in `.env` sets it at build; `site.url` in `nuxt.config.ts` is the fallback - keep the two in sync.

- **`site`** (`nuxt.config.ts`) - canonical url, name, description, `defaultLocale`. Feeds sitemap/robots/schema-org.
- **`@nuxtjs/sitemap`** - single sitemap at `/sitemap.xml` listing only this domain's homepage (explicit `urls` entry, priority 1 + curated image). `excludeAppSources: ['nuxt:route-rules']` keeps the redirect routes out; `discoverImages: false` stops it scraping optimized/external `<img>` variants. Don't add a cross-domain `sources` pull of app.earth-app.com - it leaks the app's internal routes here and Google ignores cross-domain sitemap entries anyway.
- **`@nuxtjs/robots`** - allow-all group; the `Sitemap:` line is auto-linked from the sitemap module (don't also set `robots.sitemap` or it double-lists). `mergeWithRobotsTxtPath: false` keeps config the single source of truth. AI bots are intentionally left allowed for discoverability - flip `blockAiBots` / `blockNonSeoBots` if that changes.
- **`nuxt-schema-org`** - `identity` is the Organization (logo, `sameAs`, description). WebSite + SoftwareApplication (MobileApplication/WebApplication, free `offers`) are declared in `app.vue` so they apply site-wide. The JSON-LD graph intentionally has two Organization nodes: `#identity` (graph identity) and a minimal `#organization` that carries the logo for Google's logo rich result - this is by-design, not a duplicate to "fix".

Don't fabricate schema-org data (ratings/reviews, founding dates) - only add fields backed by real facts, or Google flags the structured data.

`public/_robots.txt` is the module's backup of a former `public/robots.txt` (it renames the file on first build). With `mergeWithRobotsTxtPath: false` it's inert and can be deleted; the real `/robots.txt` is generated by the module.
